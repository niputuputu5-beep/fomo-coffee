LAPORAN AUDIT DAN PERBAIKAN PROJECT FOMO COFFEE POS
Tanggal update: 19 Mei 2026

Ringkasan:
Catatan sisa dari audit sebelumnya sudah diperbaiki. Project sekarang lebih sinkron dengan tugas.txt: Kitchen Display tersambung ke transaksi database, modul Membership/Purchase Order/Receiving memiliki schema dan API sendiri, approval flow tersedia, idle logout aktif, device/session monitoring tersedia, fitur POS placeholder sudah menjadi simulasi operasional, seed dibuat lebih aman dijalankan ulang, dan bundle frontend sudah dipecah dengan lazy loading.

Validasi teknis terakhir:
1. npm run lint: BERHASIL.
2. npm run check: BERHASIL.
3. npm run build: BERHASIL.
4. Warning chunk besar Vite sebelumnya sudah hilang. Build sekarang menghasilkan chunk per halaman/fitur.
5. npm run dev: BERJALAN di http://localhost:3000/.
6. Review bug lanjutan: BERHASIL, beberapa bug logika tambahan sudah diperbaiki.

Eksekusi database:
1. npm run db:push: BERHASIL pada 19 Mei 2026.
   Catatan: Drizzle meminta konfirmasi karena menambah unique constraint pada tabel yang sudah berisi data. Opsi yang dipilih adalah tidak melakukan truncate, sehingga data lama tetap dipertahankan.
2. npx tsx db/seed.ts: BERHASIL pada 19 Mei 2026.
   Output seed mencakup users, categories, products, product variants, customers, suppliers, inventory, promos, membership programs, purchase orders, dan receiving records.

Perbaikan catatan sisa:
1. Kitchen Display tidak lagi memakai demo state lokal.
   Perbaikan:
   - Ditambahkan endpoint transaction.kitchenQueue.
   - Kitchen Display membaca transaksi pending/processing/ready dari database.
   - Kitchen Display auto-refresh setiap 5 detik.
   - Tombol Process, Ready, dan Complete memperbarui status transaksi di database.
   - Transaksi dari POS sekarang dibuat dengan status pending agar masuk antrean kitchen.

2. Modul Membership, Purchase Order, dan Receiving tidak lagi generic-only.
   Perbaikan:
   - Ditambahkan tabel membership_programs.
   - Ditambahkan tabel purchase_orders.
   - Ditambahkan tabel receiving_records.
   - Ditambahkan API membership, purchase, dan receiving.
   - Halaman /membership, /purchase-orders, dan /receiving sekarang membaca/menulis data database.
   - Receiving otomatis membuat catatan inventory dari barang yang diterima.

3. Approval flow sudah dibuat lebih nyata.
   Perbaikan:
   - Ditambahkan tabel approval_requests.
   - Ditambahkan API approval.request, approval.list, dan approval.decide.
   - Purchase Order baru otomatis masuk pending approval.
   - Owner dapat approve Purchase Order.
   - Notification Center menampilkan approval queue.
   - Approval refund/void dapat mengubah status transaksi setelah disetujui.

4. Session timeout/idle logout sudah di-enforce di frontend.
   Perbaikan:
   - useAuth sekarang memasang idle timer 30 menit.
   - Timer reset saat mouse/keyboard/touch/scroll/click.
   - Jika fitur "Auto logout idle" dimatikan di Settings, enforcement idle logout ikut nonaktif.

5. Placeholder fitur enterprise sudah diperkuat.
   Perbaikan:
   - Device/session monitoring ditambahkan melalui tabel device_sessions dan API device.
   - Layout mengirim heartbeat device saat user masuk ke aplikasi.
   - Settings menampilkan daftar device/session dan tombol revoke aktif.
   - POS memiliki offline mode lokal: saat browser offline, transaksi disimpan ke localStorage queue.
   - POS memiliki tombol print receipt dengan window.print().
   - POS memiliki tombol open cash drawer berupa command simulation/toast.
   - POS memiliki tombol WhatsApp customer shortcut.
   - Kitchen Display memakai polling server sebagai real-time sync sederhana.
   - Settings memiliki toggle OTP optional, offline mode, real-time sync, dan WhatsApp integration.

6. Seed database dibuat lebih idempotent.
   Perbaikan:
   - Seed memakai insert ignore untuk data demo.
   - Ditambahkan unique key pada field demo penting: product variant sku, customer phone, supplier name, inventory sku, promo code, membership program name.
   - Seed ditambah data demo membership, purchase order, dan receiving.
   - Menjalankan seed berkali-kali tidak lagi menggandakan data demo utama yang memiliki key unik.

7. Bundle frontend besar sudah dioptimasi.
   Perbaikan:
   - App routing memakai React.lazy dan Suspense.
   - Halaman besar seperti POS, Reports, Settings, Dashboard, Kitchen, dan ModulePage dipisah menjadi chunk sendiri.
   - Build final tidak lagi memberi warning chunk > 500 kB.

8. Responsiveness halaman diperbaiki.
   Perbaikan:
   - Sidebar aplikasi berubah menjadi drawer di layar mobile, dengan overlay dan tombol buka/tutup.
   - Header aplikasi dibuat lebih ringkas di mobile.
   - Padding konten disesuaikan antara mobile dan desktop.
   - Semua tabel utama diberi horizontal scroll otomatis agar tidak memecah layout di mobile/tablet.
   - POS Kasir berubah dari layout dua kolom kaku menjadi stack di mobile/tablet, lalu dua kolom lagi di desktop besar.
   - Product grid POS memakai 1 kolom di layar sangat kecil, 2 kolom di mobile lebar, dan meningkat di desktop.
   - Action bar/filter pada Produk, Customer, Inventory, Supplier, Promo, Transaksi, Kitchen, Reports, Shifts, User Management, ModulePage, dan Settings dibuat wrap/stack di mobile.
   - Card informasi panjang seperti supplier address, email customer, low stock, dan footer promo dibuat truncate/stack agar tidak melebar.

9. Bug logika tambahan dari review ulang diperbaiki.
   Perbaikan:
   - Endpoint update status transaksi sekarang memvalidasi transaksi ada, membatasi kasir hanya pada transaksi miliknya sendiri, dan mencegah void langsung selain oleh Owner.
   - Promo POS sekarang hanya memberi diskon setelah kode promo berhasil divalidasi, masih aktif, belum expired, memenuhi minimum transaksi, dan belum melewati usage limit.
   - Mengubah teks kode promo otomatis membatalkan promo yang sebelumnya diterapkan, sehingga diskon tidak nyangkut.
   - Pencarian produk sekarang tetap menyaring produk aktif saja.
   - Transaksi POS sekarang mengurangi stockQuantity produk secara otomatis.
   - Closing shift tidak lagi menimpa breakdown pembayaran menjadi nol.
   - getOpenShift sekarang selalu mengambil shift aktif milik user yang sedang login agar POS tidak salah membaca shift user lain.
   - Toggle status user tidak lagi menghapus email user.
   - Forgot password sekarang membutuhkan username dan email akun yang cocok, tidak hanya username.
   - Approval yang sudah diputus tidak diproses ulang.
   - Approval Purchase Order dibatasi ke entityType purchase_order saat update approval request.

   - Ditambahkan totalCogs dan grossProfit pada transaksi.

   - Checkout POS menghitung HPP per item dari resep bahan baku x unitCost inventory.
   - Jika produk belum punya resep, sistem memakai costPrice produk sebagai fallback HPP.
   - Checkout POS mengurangi stok produk dan juga mengurangi stok bahan baku sesuai resep.
   - Laporan salesSummary sekarang mengembalikan totalCogs, grossProfit, dan grossMargin.
   - Top products sekarang membawa revenue, cogs, grossProfit, dan margin.
   - Halaman Reports menampilkan HPP/COGS dan Gross Profit.
   - Seed database menambahkan resep realistis untuk produk demo coffee shop.

   Perbaikan:

   - Grafik penjualan harian dashboard sekarang dihitung dari transaksi hari ini.
   - Low stock dashboard sekarang menampilkan empty state jika tidak ada stok menipis, bukan data bahan palsu.
   - Promo page memperbaiki label jumlah dari "promo aktif" menjadi "promo terdaftar" karena list menampilkan semua status.
   - Promo page memperbaiki teks "No expiry" menjadi "Tanpa batas tanggal".
   - Promo page mengganti simbol infinity menjadi teks ASCII "unlimited" agar tidak rusak encoding.

20. Perbaikan bug layout POS: menu tidak muncul dan panel pembayaran tidak bisa discroll.
   Temuan:
   - Panel kanan POS memakai overflow-hidden pada mode desktop, sehingga isi seperti metode bayar, uang diterima, catatan, dan tombol Complete Order bisa terpotong saat tinggi layar terbatas.
   - Area menu belum memiliki state loading/kosong yang jelas, sehingga saat query produk belum selesai atau hasil filter kosong terlihat seperti menu hilang.
   Perbaikan:
   - Layout POS diganti menjadi grid desktop dengan kolom menu fleksibel dan panel pembayaran 400px yang stabil.
   - Panel pembayaran dibuat sticky dan scrollable di desktop, sehingga seluruh isi bisa digulir sampai tombol Complete Order.
   - Area menu diberi min-height dan batas scroll desktop agar daftar produk tetap terlihat dan tidak terdorong oleh panel kanan.
   - Ditambahkan skeleton loading produk dan empty state "Menu tidak ditemukan" untuk memperjelas kondisi data kosong/filter tidak cocok.
   - Tinggi tombol Complete Order distabilkan ke h-12 agar class Tailwind valid dan konsisten.

21. Perbaikan add-ons POS kasir dan bug HPP item duplikat.
   Temuan:
   - Add-ons di POS sebelumnya hanya berupa chip kecil di bawah item keranjang, dibatasi 6 item, dan tidak muncul saat kasir memilih produk dari menu.
   - Kasir tidak punya alur jelas untuk memilih tambahan kopi, es batu, saus, atau add-ons lain sebelum item masuk keranjang.
   - Jika produk yang sama masuk beberapa kali dengan konfigurasi/add-ons berbeda, backend menghitung HPP item berdasarkan productId pertama yang cocok, sehingga gross profit item bisa tidak sinkron.
   Perbaikan:
   - Klik produk di POS sekarang membuka dialog konfigurasi item.
   - Dialog POS menampilkan level gula, level es, seluruh add-ons aktif, tombol plus/minus kuantitas add-ons, reset add-ons, dan preview harga item.
   - Add-ons bisa diedit ulang dari item keranjang melalui tombol Edit add-ons.
   - Harga item, subtotal keranjang, payload transaksi, HPP add-ons, dan pengurangan inventory add-ons tetap ikut tersambung.
   - Backend transaction.create menghitung HPP item berdasarkan index item, bukan hanya productId, agar produk sama dengan add-ons berbeda tetap akurat.

   - Mode Opname cepat memungkinkan input stok aktual langsung di tabel, melihat selisih otomatis, menulis catatan, lalu simpan opname per item.
   - Riwayat Stok Opname sekarang punya filter item dan tipe adjustment.
   - Search inventory backend sekarang mencari nama item, SKU, dan lokasi.

   Temuan:
   - Popup option pada native select tetap memakai UI browser/Windows sehingga masih biru dan kaku saat dropdown dibuka.

   - Ditambahkan komponen ThemedSelect berbasis Radix Select dengan popup option bertema FOMO.

   - baseUrl di tsconfig.app.json dihapus karena paths tetap berjalan tanpa baseUrl pada moduleResolution bundler, sehingga warning deprecated hilang.
