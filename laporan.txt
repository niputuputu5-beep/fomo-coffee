LAPORAN PROJECT FOMO COFFEE POS
Tanggal update: 22 Mei 2026

1. Ringkasan Project

FOMO COFFEE POS adalah aplikasi Point of Sales coffee shop berbasis React, TypeScript, Vite, Hono, tRPC, Drizzle ORM, dan MySQL. Project ini dibuat sebagai sistem operasional coffee shop dengan role Owner, Admin, dan Kasir, serta modul POS, kitchen display, produk, kategori, inventory, supplier, customer, membership, promo, purchase order, receiving, transaksi, shift, laporan, audit log, user management, notification center, dan halaman publik.

2. Audit Kekurangan Fitur dan Integrasi

Hasil pengecekan project pada 22 Mei 2026:

1. Fitur tambah/edit produk belum bisa input gambar dari user.
   - Kondisi sebelumnya: form produk tidak memiliki input gambar.
   - Kondisi sebelumnya: produk baru selalu memakai gambar default /products/cappuccino.jpg.
   - Dampak: admin tidak bisa upload foto produk dari komputer dan tidak bisa memakai URL gambar eksternal.

2. Integrasi upload file belum tersedia di backend.
   - Kondisi sebelumnya: tidak ada endpoint API untuk menerima file gambar.
   - Dampak: frontend tidak punya jalur aman untuk menyimpan file upload.

3. Static serving untuk file upload runtime belum tersedia.
   - Kondisi sebelumnya: server hanya melayani asset build dan public asset bawaan.
   - Dampak: jika file berhasil disimpan secara manual, file belum tentu bisa diakses dari browser.

4. Validasi keamanan upload belum tersedia.
   - Kondisi sebelumnya: tidak ada validasi role, ukuran file, dan MIME type karena fitur upload belum ada.
   - Dampak: jika upload dibuat tanpa kontrol, fitur berisiko menerima file non-gambar atau file terlalu besar.

5. Bug form produk saat field flag kosong.
   - Kondisi sebelumnya: values.flags.toLowerCase() dapat error jika field flag tidak terisi.
   - Dampak: submit produk bisa gagal karena input opsional kosong.

6. Catatan integrasi eksternal yang masih perlu provider produksi:
   - OTP masih perlu integrasi email/SMS/Authenticator.
   - Thermal printer dan cash drawer masih perlu driver/bridge perangkat fisik.
   - WhatsApp masih berupa shortcut/placeholder, belum memakai WhatsApp Business API.
   - Penyimpanan upload saat ini memakai filesystem aplikasi. Untuk produksi Railway jangka panjang, lebih ideal memakai object storage seperti S3/R2 karena filesystem Railway dapat hilang saat redeploy/rebuild.

3. Perbaikan yang Dilakukan

1. Menambahkan input gambar pada form produk.
   - Admin/Owner sekarang bisa mengisi URL gambar produk.
   - Admin/Owner sekarang bisa upload gambar dari komputer.
   - Format yang diterima: JPG, PNG, WEBP, GIF.

2. Menambahkan endpoint upload gambar produk.
   - Endpoint: POST /api/uploads/product-image
   - Request: multipart/form-data dengan field image.
   - Response sukses: { url: "/uploads/product-images/nama-file.ext" }

3. Menambahkan proteksi upload.
   - Upload hanya bisa dilakukan user yang sudah login.
   - Upload hanya boleh untuk role owner dan admin.
   - Ukuran file maksimal 5 MB.
   - File selain JPG, PNG, WEBP, dan GIF ditolak.

4. Menambahkan serving file upload.
   - File upload disimpan ke folder uploads/product-images.
   - File dapat diakses melalui URL /uploads/product-images/....

5. Memperbaiki submit produk.
   - Produk baru tidak lagi selalu dipaksa memakai cappuccino jika user memberi URL/upload.
   - Edit produk mempertahankan gambar lama jika user tidak mengubah gambar.
   - Field flag kosong tidak lagi membuat form error.

6. Memperluas komponen QuickFormDialog.
   - QuickFormDialog sekarang mendukung field type file.
   - Existing form lain tetap memakai behavior lama.

7. Struktur Project

Struktur utama:

- api/
  Backend Hono, tRPC router, middleware, auth, session, audit, env, dan helper database.

- db/
  Drizzle schema, relation, dan seed data.

- src/
  Frontend React, halaman, komponen layout, komponen UI, provider tRPC, hooks, dan styling.

- contracts/
  Constant dan tipe bersama frontend-backend.

- public/
  Asset publik bawaan seperti logo dan gambar produk demo.

- uploads/
  Folder runtime untuk gambar produk hasil upload.

- scripts/
  Script start production.

8. Step Instalasi Lokal dari Awal

1. Install prasyarat:
   - Node.js minimal 20.19.0.
   - MySQL 8 atau MariaDB.
   - npm.

2. Clone atau buka folder project:
   cd D:\fomo-coffee

3. Install dependency:
   npm install

4. Buat database MySQL:
   CREATE DATABASE fomo_coffee;

5. Buat file .env:
   Copy-Item .env.example .env

6. Isi .env:
   DATABASE_URL=mysql://root:@localhost:3306/fomo_coffee
   SESSION_SECRET=fomo-coffee-local-development-secret

   Jika MySQL memakai password:
   DATABASE_URL=mysql://root:password_kamu@localhost:3306/fomo_coffee

7. Push schema ke database:
   npm run db:push

8. Isi data demo:
   npx tsx db/seed.ts

9. Jalankan development server:
   npm run dev

10. Buka aplikasi:
   http://localhost:3000

10. Pemasangan Database

Database yang digunakan adalah MySQL.

Langkah:

1. Pastikan MySQL aktif.
2. Buat database:
   CREATE DATABASE fomo_coffee;
3. Pastikan DATABASE_URL di .env sesuai username, password, host, port, dan nama database.
4. Jalankan:
   npm run db:push
5. Jalankan seed:
   npx tsx db/seed.ts

Command penting database:

- npm run db:push
  Menyesuaikan schema Drizzle ke database.

- npm run db:generate
  Membuat file migration dari perubahan schema jika ingin memakai migration file.

- npm run db:migrate
  Menjalankan migration yang sudah dibuat.

- npx tsx db/seed.ts
  Mengisi data demo.

11. Deploy ke Railway

A. Persiapan Railway

1. Login ke Railway.
2. Buat project baru atau pakai project FOMO COFFEE yang sudah ada.
3. Hubungkan repository GitHub project ini.
4. Tambahkan service database MySQL di Railway atau gunakan database eksternal.
5. Salin connection string MySQL ke environment variable DATABASE_URL.

B. Environment Variable Railway

Tambahkan variable:

NODE_ENV=production
DATABASE_URL=mysql://user:password@host:port/database
SESSION_SECRET=isi-dengan-secret-panjang-random

C. Build dan Start Command

Project sudah memiliki railway.json:

Build command:
npm run build

Start command:
npm run start

D. Push Schema Database Railway

Pilihan 1, dari lokal:

1. Set DATABASE_URL lokal sementara ke connection string Railway.
2. Jalankan:
   npm run db:push
3. Jika perlu data demo:
   npx tsx db/seed.ts
4. Kembalikan .env lokal ke database lokal.

Pilihan 2, dari Railway shell jika tersedia:

1. Buka shell service aplikasi.
2. Jalankan:
   npm run db:push
3. Jika perlu:
   npx tsx db/seed.ts

E. Deploy

1. Commit perubahan:
   git add .
   git commit -m "feat: add product image upload"

2. Push ke GitHub:
   git push origin main

3. Railway otomatis deploy jika service terhubung ke branch main.

4. Jika ingin deploy manual memakai Railway CLI:
   railway login
   railway link
   railway up

5. Setelah deploy, cek halaman:
   - /login
   - /products
   - /pos
   - /dashboard

13. Catatan Produksi

1. Untuk Railway produksi, upload gambar berbasis filesystem cukup untuk demo, tetapi belum ideal untuk penyimpanan permanen lintas redeploy.
2. Rekomendasi produksi berikutnya adalah menambahkan object storage S3/R2 dan menyimpan URL object storage ke field products.image.
3. SESSION_SECRET wajib diganti dengan string panjang acak.
4. DATABASE_URL produksi jangan disimpan di repository.
5. Jalankan npm run lint, npm run check, npm test, dan npm run build sebelum push.

14. Audit Ulang Fitur, Layout, dan Sistem - 22 Mei 2026

Temuan yang diperbaiki:

1. Form produk masih bisa error jika field URL gambar tidak tersedia.
   - Perbaikan: normalisasi URL gambar sekarang aman untuk nilai kosong/undefined.

2. Dialog form cepat bisa terlalu tinggi di layar kecil.
   - Perbaikan: QuickFormDialog diberi max-height 92vh dan overflow scroll.

3. Beberapa tabel besar masih melebar keluar layout mobile.
   - Perbaikan: tabel User Management, Reset Password Request, Transaksi, Audit Log, dan Device Session diberi wrapper overflow-x-auto serta min-width yang stabil.

4. Tombol reprint pada list transaksi mencetak halaman terlalu cepat sebelum detail transaksi siap.
   - Perbaikan: tombol tersebut sekarang membuka detail transaksi; proses reprint dilakukan dari dialog detail.

5. Dialog detail transaksi bisa kepanjangan di viewport kecil.
   - Perbaikan: dialog diberi max-height dan scroll internal.

6. HTML struk POS memakai data transaksi langsung.
   - Perbaikan: data dinamis pada HTML struk sekarang di-escape sebelum ditulis ke popup print.

7. Tombol WhatsApp POS bisa membuka link tanpa nomor customer.
   - Perbaikan: sistem sekarang mengecek nomor customer dan memberi toast error jika nomor belum tersedia.

8. Empty state Kitchen Display hanya mengisi satu kolom pada layar besar.
   - Perbaikan: empty state dibuat span penuh sesuai jumlah kolom grid.

9. Laporan penjualan masih berisiko menghitung transaksi void, cancelled, refunded, atau payment void.
   - Perbaikan: sales summary, sales trend, hourly sales, payment distribution, dan top products hanya menghitung transaksi reportable.
